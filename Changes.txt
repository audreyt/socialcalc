==============================================================================
v3.2.0 — 2026-07-21
==============================================================================

Highlights
- Interoperability now covers BOM-safe CSV/TSV input, csv-eu input,
  csv-excel/tab-excel output, normalized-workbook ingestion, and FODS export.
- Multi-sheet workbook operations, cross-sheet recalculation, workbook
  save/load, AutoFilter, structured tables, pivot tables, charts, data
  validation, sheet protection, conditional formatting, and rich HTML-table
  clipboard paste are documented in this release.
- Added dynamic arrays and new formula families including RANK/MEDIAN/QUARTILE,
  statistics, weekday/workday, criteria, and XMATCH/XLOOKUP functions, plus
  LET/LAMBDA and higher-order array functions.
- Find remains globally visible and right-aligned; the replacement controls use
  a progressive right-aligned row.
- Added print/page-setup metadata and a Print toolbar tab, an ARIA-accessible
  grid, and formal-verification growth from 48 to 195 Dafny VCs across the
  release's expanded LemmaScript facade set.

CSV/TSV import/export
- UTF-8 BOM is now stripped from csv/tab/csv-eu import input (fixes corrupted
  first cell on every Excel-exported UTF-8 CSV). Default csv/tab output is
  byte-identical to prior behavior.
- New "csv-eu" input format: ";" field delimiter + "," decimal point
  (European/continental comma-decimal convention). Legacy "csv"/"tab" are
  completely unaffected. Uses an explicit fixed-locale policy (NOT an
  autodetecting tie-break) — under csv-eu, "," is always the decimal, "."
  is always thousands grouping, never guessed from position. A comma-bearing
  token that fails locale numeric parsing becomes text rather than silently
  reinterpreted under US-locale comma-thousands rules.
- New "csv-excel"/"tab-excel" output formats: BOM prefix + CRLF row
  terminators for Windows/Excel-friendly export. Default "csv"/"tab" output
  is byte-identical (no BOM, LF only).

Normalized-workbook ingestion seam
- `SC.CreateSheetSaveFromNormalizedSheet(normalizedSheet, skipped?)`: maps a
  plain, already-parsed, TRUSTED structured JSON object into a SocialCalc
  native scsave string using existing Sheet/cell/style primitives
  (SetConvertedCell, GetStyleNum). No binary/zip/XML format parsing — this
  is the seam a HOST-SUPPLIED external parser (XLSX/ODS reader in the host
  adapter or EtherCalc server) targets. Skipped coord/name keys are
  collected in the optional `skipped` array as "cell:B7"/"name:1BAD".
- `SC.CreateSpreadsheetSaveFromNormalizedWorkbook(normalizedWorkbook)`: maps
  multiple sheets at once, de-duplicating sheet names, returning one scsave
  string per sheet keyed by display name.
- Optional per-sheet `formulaSeparator: ";"` flag normalizes semicolon-
  authored formula text to SocialCalc's native comma-separated syntax
  BEFORE parsing — quote-aware, never touches the shared tokenizer/parser.
  Documented hard precondition: decimal commas must already be "." before
  calling with this flag (non-recoverable ambiguity documented in source).

FODS (OpenDocument Flat XML Spreadsheet) export
- `SC.CreateFodsFromNormalizedWorkbook(normalizedWorkbook)`: emits a
  standards-valid ODF 1.2 flat-XML spreadsheet document (no ZIP/deflate
  dependency). Encodes cell values (numbers, text), formulas (translated
  to OpenFormula "of:" canonical form via the shared formula tokenizer:
  cell refs → "[.A1]", ranges → "[.A1:.B2]", "," → ";"), cached formula
  results (office:value), bold/italic/alignment cell styles (deduped via
  <office:automatic-styles>), cell comments (office:annotation), and
  named ranges (table:named-range with CellAddress/CellRangeAddress grammar).
  Verified by real LibreOffice 26.2.4.2 `soffice --convert-to fods`
  round-trip (formula, cached value, named-range address, and element order
  all preserved). Sheet names requiring quoting are single-quoted per ODF
  QuotedSheetName. Named-range definitions are validated (coord-shaped)
  before emission; malformed defs are skipped rather than producing broken
  cell-range-address values.

Formula functions and dynamic arrays
- Added 107 registered formula names plus the `LET` and `LAMBDA` special
  forms. Every new formula name is enumerated below by family.
- Completed the dynamic-array implementation for
  [audreyt/ethercalc#650](https://github.com/audreyt/ethercalc/issues/650).
  `SORT`, `UNIQUE`, `FILTER`, `SEQUENCE`, `TRANSPOSE`, `SORTBY`,
  `CHOOSECOLS`, `CHOOSEROWS`, `TAKE`, `DROP`, `HSTACK`, `VSTACK`, `TOCOL`,
  `TOROW`, `WRAPROWS`, `WRAPCOLS`, and `EXPAND` return typed arrays through
  one transactional spill pipeline. Spill ownership, collision handling,
  recalc grow/shrink, persistence, command guards, and editor integration
  prevent partial or stale arrays.
- Added lexical `LET` and `LAMBDA` expressions, named closures, nested scope
  and recursion limits, plus the higher-order array functions `MAP`, `REDUCE`,
  `SCAN`, `BYROW`, `BYCOL`, and `MAKEARRAY`.
- Added `RANK(number,ref,[order])`, `MEDIAN(v1,c1:c2,...)`, and
  `QUARTILE(range,quart)` for audreyt/ethercalc#712 and #726, together with
  `RANK.EQ`, `RANK.AVG`, `QUARTILE.INC`, `QUARTILE.EXC`, `PERCENTILE`,
  `PERCENTILE.INC`, `PERCENTILE.EXC`, `PERCENTRANK`, `PERCENTRANK.INC`, and
  `PERCENTRANK.EXC`. `RANK` is descending by default, tied values share the
  best rank, and a missing value is `#N/A`; `RANK.AVG` averages tied ranks.
  `MEDIAN` averages the middle pair and returns `#DIV/0!` when no numeric
  input remains. Inclusive quartiles truncate the quart argument toward zero
  before validating 0..4; invalid or empty quartiles return `#NUM!`. The
  family shares numeric-only extraction, blank/text skipping, and
  first-range-error propagation. The exact rank/quartile boundary policies
  are verified by the 3-VC `lemma/statistics.ts` facade.
- Added math and modern statistical compatibility functions `SIGN`, `GCD`,
  `LCM`, `MROUND`, `QUOTIENT`, `ROUNDUP`, `ROUNDDOWN`, `SUMSQ`, `STDEV.P`,
  `STDEV.S`, `VAR.P`, and `VAR.S`.
- Added paired-range statistics `CORREL`, `COVARIANCE.P`, `COVARIANCE.S`,
  `SLOPE`, `INTERCEPT`, `RSQ`, `FORECAST`, `FORECAST.LINEAR`, `TREND`, and
  `GROWTH`, with equal-shape validation and shared numeric/error handling.
- Added `SUBTOTAL`, including function-code validation and filtered/manual
  hidden-row semantics shared with AutoFilter.
- Added date and workday functions `EDATE`, `EOMONTH`, `DATEDIF`, `WEEKNUM`,
  `ISOWEEKNUM`, `YEARFRAC`, `WORKDAY`, `WORKDAY.INTL`, `NETWORKDAYS`, and
  `NETWORKDAYS.INTL`, including holiday ranges and Excel-compatible weekend
  codes/masks.
- Added financial functions `PPMT`, `IPMT`, `MIRR`, `XNPV`, and `XIRR`,
  including irregular dated cash flows and bounded root solving.
- Added reference, information, and rendering functions `ROW`, `COLUMN`,
  `ADDRESS`, `INDIRECT`, `OFFSET`, `ISFORMULA`, `ISREF`, `ERROR.TYPE`, `TYPE`,
  `HYPERLINK`, `IMAGE`, `TEXT`, and `SEARCH`. Dynamic references participate
  in recalculation dependencies; rendered URLs remain subject to the
  untrusted-content URL policy.
- Added logical/error functions `IFERROR`, `IFNA`, `IFS`, and `SWITCH`.
- Added text/array functions `TEXTJOIN`, `JOIN`, `TEXTBEFORE`, `TEXTAFTER`,
  `SPLIT`, and `TEXTSPLIT`, plus `REGEXMATCH`, `REGEXEXTRACT`, and
  `REGEXREPLACE`.
- Added `XMATCH(lookup_value,lookup_array,[match_mode],[search_mode])` and
  `XLOOKUP(lookup_value,lookup_array,return_array,[if_not_found],[match_mode],
  [search_mode])`. `match_mode` supports exact (0), exact-or-next-smaller
  (-1), exact-or-next-larger (1), and wildcard (2, with `*`, `?`, and `~`
  escaping). `search_mode` supports first-to-last (1), last-to-first (-1),
  and binary ascending/descending (2/-2, assuming sorted input); binary
  search with wildcard matching is `#VALUE!`. Lookup and return vectors must
  be single rows or columns of equal axis length. Misses are `#N/A` unless
  `if_not_found` is supplied, and matched 2-D rows/columns spill through the
  dynamic-array pipeline. The 7-VC `lemma/xlookup.ts` facade verifies mode
  legality and exact-before-approximate-before-miss precedence.
- Added `COUNTIFS`, `AVERAGEIF`, `AVERAGEIFS`, `MAXIFS`, and `MINIFS` without
  changing the legacy `COUNTIF`/`SUMIF`/`SUMIFS` range-walk behavior. Every
  criteria and aggregate range in the new family must have the same shape;
  cross-sheet and named ranges are supported. Matched errors propagate,
  average functions return `#DIV/0!` for no numeric matches, and
  `MAXIFS`/`MINIFS` return zero for no matches. The 5-VC `lemma/criteria.ts`
  facade verifies row eligibility and result/error precedence.

Spreadsheet capabilities
- Added optional workbook/multi-sheet mode with add, delete, rename, reorder,
  hide, and unhide operations; accessible sheet tabs; cross-sheet
  recalculation; undo/redo; and a versioned workbook save/load envelope.
- Added AutoFilter and structured tables with per-column criteria, composed
  manual/filter row visibility, dropdown value collection, structural range
  adjustment, persistence, and undo/redo. `SUBTOTAL` consumes the same
  visibility policy.
- Added pivot tables with row/column groups, filters, `SUM`/`COUNT`/`COUNTA`/
  `AVERAGE`/`MIN`/`MAX` aggregation, subtotals and grand totals, transactional
  output ownership, refresh/recalc, structural adjustment, persistence, UI,
  and undo/redo.
- Added anchored column, bar, line, area, and pie charts with safe SVG
  rendering, range-backed series, titles/labels/legends, editor dialogs,
  drag/resize/delete controls, structural reference adjustment, persistence,
  and undo/redo.
- Added cell data validation for lists, numbers, dates, text length, and
  custom formulas, with reject/warn/allow-blank policies, safe dropdowns,
  edit/paste enforcement, coordinate-aware rewrites, persistence, and
  undo/redo.
- Added sheet protection with per-cell unlocked exceptions. Protected cells
  reject edit, erase, fill, paste, merge, and sort operations while unlocked
  cells remain editable; the state round-trips through save/load and
  undo/redo.
- Added conditional formatting with ordered, stop-if-true rules for
  comparisons, text, blank/nonblank, duplicate/unique, and custom formulas.
  Relative formulas adjust across copy/fill/structural edits; safe palette
  styles render without mutating base cell styles and persist with undo/redo.
- Added rich HTML-table clipboard paste with sanitized text, rowspan/colspan
  expansion, and an allowlisted formatting subset (bold, italic, alignment,
  foreground/background colors), applied as one undoable paste transaction.

Editor productivity
- Refined Find/Replace: Find remains globally visible and right-aligned on the
  formula row; replacement controls appear only for a non-empty query in their
  own right-aligned row, dynamically resizing editor chrome and the grid.
  Queries and replacements persist across tool tabs, and clearing the query
  restores the prior layout.
- Fixed pane-slider freezing to retain the visible rows and columns, with
  unfreeze restoring the original pane boundaries.

Printing and page setup
- Added print/page-setup sheet metadata: `printarea`, `printrepeatrows`,
  `printrepeatcols`, `printorientation`, `printscale`, and `printmargins` are
  new sheet attributes, set via `set sheet print...` commands (undo-able,
  save/load round-trippable, backward-compatible — sheets without print
  settings never emit the new save tokens). A minimal "Print" toolbar tab
  lets a host configure these fields and trigger `window.print()` from a
  real button click; `SocialCalc.PreparePrintArea` marks the rendered grid
  and injects a real `@page` CSS rule (orientation/margins/scale) so
  `@media print` hides editor chrome, prints only the configured area, and
  repeats header rows across pages as CSS `table-header-group` permits.

Accessibility
- Added real ARIA accessibility to the rendered grid: `role="grid"` with
  `aria-rowcount`/`aria-colcount`, `role="row"`/`"rowheader"`/
  `"columnheader"`/`"gridcell"` with row/column indices and plain-text
  accessible names, `aria-selected`/`aria-readonly`/`aria-invalid` kept in
  sync on every keyboard/mouse cursor move (not just full re-render), a
  keyboard-focusable grid (`tabindex="0"`, `aria-activedescendant`), and
  visually-hidden `aria-live` regions announcing cell navigation and
  command errors. All ARIA attributes are set via `setAttribute`/
  `textContent` from plain cell data, never by concatenating sheet content
  into markup.

Formal verification
- Expanded the LemmaScript proof surface with facades for dynamic-array spill,
  weekday policy, statistics, visibility, XMATCH/XLOOKUP, protection,
  conditional formatting, workbooks, charts, validation, LET/LAMBDA scope,
  pivot tables, criteria aggregation, and HTML-table clipboard policy.
- The release now documents 20 proof-bearing facades and 195 total Dafny VCs,
  up from the prior 48; the original A1, evaluation-operations, lookup-result,
  and number-parse facades remain part of that total.

v3.1.0 — 2026-07-14
==============================================================================

Highlights
- Added an opt-in rendering policy for untrusted workbooks while preserving
  byte-compatible legacy rendering by default. Hosts can enable
  `SocialCalc.Callbacks.untrustedContent`, supply an HTML sanitizer, and
  configure URL-scheme and `data:` MIME allowlists.
- Added `dist/SocialCalc.min.js` as a supported browser/CommonJS distribution
  artifact. The package now declares its CommonJS type explicitly and supports
  Node.js 22 and 24 as its CI-tested runtime range (`engines.node` is `>=22`).
- Replaced assumed release quality with executable gates: tarball-first package
  checks, source-attributed unit and browser coverage, real-browser tests,
  differential/adversarial/performance tests, test-credibility checks, mutation
  floors for all eleven shipping modules, a pinned EtherCalc canary, and a
  deterministic, provenance-aware release pipeline.
- Added a Vite+ root development workbench with local jQuery and bundle assets
  and an editable spreadsheet for browser-based development.
- SORT now accepts omitted sort columns and directions, defaults to the first
  column ascending, and treats non-positive directions as descending.
- Added editor productivity features: Find & Replace (literal and regex
  modes, selected-range or whole-sheet scope, formula preservation by
  default with an explicit "Include formulas" opt-in, one-undo
  transaction for Replace-All), discoverable Freeze/Unfreeze Panes
  toolbar buttons wired to the existing pane row/col commands, Ctrl+Y
  and Ctrl+Shift+Z redo across all three browser keyboard maps, a
  merged-cell guard that rejects sort over a range containing any
  merged cell without corrupting spans, and a modern
  `navigator.clipboard.writeText`/`readText` path for copy/paste with
  the pre-existing focused-textarea fallback preserved unchanged when
  the modern API is unavailable or rejected.


Compatibility and runtime fixes
- `EvaluatePolish` now distinguishes genuine NaN results from numeric overflow
  before normalizing the computed value, restoring the correct NaN error text.
- `RecalcLoadedSheet` no longer throws when neither an explicit sheet name nor
  a queued cache name is available.
- `InitializeSpreadsheetControl` returns after reporting a missing parent node
  instead of falling through and assigning the null parent.
- Popup dispatch now treats an unknown or handler-less type as a no-op, matching
  the other popup dispatchers instead of throwing from `Popup.Reset`.
- Popup RGB parsing preserves signs, clamps channels to 0–255, truncates
  fractional channels consistently, and always emits two hex digits per
  channel; malformed or non-finite inputs normalize safely.
- `setemailparameters` now completes the editor busy/deferred-command cycle
  even when an `EMAILIF`/`EMAILONEDITIF` trigger sends no email or throws, while
  avoiding a duplicate completion when a real send already supplied one.
- Removed an undeclared no-op assignment in partial AM/PM format parsing that
  threw under strict Node execution without contributing to the result.
- Removed 19 provably unreachable defensive coverage records (16 branch arms
  and 3 statements): redundant numeric/type checks, repeated clamps,
  regex-impossible fallbacks, guaranteed `split`/`createElement` fallbacks,
  and an exhaustive parser tail condition.

Security
- `SocialCalc.Callbacks.untrustedContent` defaults to `false`, preserving the
  historical trusted-workbook behavior. When enabled, raw HTML formats,
  text-custom templates, wiki callback output, and other sheet-derived HTML
  sinks are escaped or passed through the host's
  `securityPolicy.sanitizeHtml` callback.
- URL rendering validates non-`data:` URLs against
  `securityPolicy.allowedUrlSchemes`; `data:` URLs use the separate
  `allowedDataMimeTypes` allowlist and are rejected by default.
- Disabled formula-widget `cell_html` rendering in untrusted-content mode
  because its sheet-authored event-capable markup cannot be safely escaped
  field by field.
- Closed an HTML entity-decoding bypass in `SafeUrlForRender`: entity-encoded
  scheme characters and quote characters can no longer become active
  `javascript:` URLs or injected attributes after browser HTML parsing.
- Added DOM-level Chromium assertions for every enumerated active-content sink
  and documented the sink inventory in `docs/security-sink-inventory.md`.

Packaging and consumer contract
- `package.json` now declares `"type": "commonjs"`. Supported entry points are
  CommonJS `require("socialcalc")`, native ESM default import, browser-global
  loading of `dist/SocialCalc.js`, and their minified-bundle equivalents.
  Native ESM named imports remain intentionally unsupported.
- `prepack` always runs `vp build --minify`; the minified bundle is generated
  and gitignored rather than hand-maintained.
- Added `vp run test:package-contract`, which packs a fresh candidate and pins
  the exact 17-member tarball allowlist. It exercises root CommonJS, deep
  minified CommonJS, native ESM default import, both browser-global bundles,
  strict external TypeScript consumption, command/formula/save-load behavior,
  delivery-path API-shape parity, and component/package size ceilings.
- Added Node 22/24 packed-tarball compatibility jobs and a release-only
  EtherCalc canary against a pinned EtherCalc checkout and lockfile.

Build, tests, and coverage
- Moved the ordered global-script build and test lifecycle to Vite+.
  `build.ts` is the Vite plugin, strips global-script TypeScript with Oxc,
  preserves source license preambles, emits the UMD bundle and CSS, and emits
  the minified UMD when requested.
- Migrated tests to `vite-plus/test`. The shared harness compiles the generated
  bundle once per isolated Vitest worker instead of creating query-expanded
  copies of the approximately 720 KB bundle.
- Added a pinned, offline `socialcalc@3.0.8` oracle and differential coverage
  for formulas, formatting, commands, references, names, save/load/export, and
  headless initialization. Added bounded malformed-input, extreme-range,
  circular-reference, prototype-key, idempotency, bundle-size, and recalc-time
  checks.
- Added deterministic Playwright coverage across Chromium, Firefox, and WebKit
  for the normal and minified UMD bundles, with page-error, rejection,
  console-error, and unexpected-dialog failures surfaced directly.
- Added `check:test-credibility`, which rejects tautological assertions and
  unexplained code-free catches across tracked Vitest and Playwright tests.
  Existing coverage-only catches were replaced with observable state, command,
  callback, DOM, and error assertions.
- Made `vp check` the non-mutating CI and release gate for formatting,
  warning-free type-aware lint, and types. Ordinary builds now canonicalize
  generated `dist/SocialCalc.js` with the same formatter, preserve the legacy
  UMD quote form required by EtherCalc's headless adapter, and verify the
  generated file; coverage builds retain exact pre-format text for source maps.
- Made Istanbul the default `vp test` coverage gate. Vitest global setup builds
  a fresh counter-instrumented UMD before workers start, and the full suite now
  enforces 100% statements, branches, functions, and lines across all eleven
  shipping `js/*.ts` modules and the three LemmaScript facades.
- Retained source-attributed V8 coverage as the explicit
  `vp run test:coverage` diagnostic and as the unit half of merged
  Vitest+Chromium coverage. The shared release floors remain
  98/80/98/98 for unit V8 and 98/84/98/98 after the browser merge.
- Added `bun run test:bun`, which runs the same 2,941-test corpus in isolated
  native Bun workers. Bun coverage is not the release gate because it does not
  collect the `vm.Script`-loaded UMD or expose statement/branch counters.

Mutation and formal verification

Release engineering and dependencies

v3.0.6 — 2026-07-09
==============================================================================


Fixes
- Formula arithmetic `/` now propagates error operands (e.g. `1/#REF!` →
  `#REF!`) instead of treating coerced error zeros as `#DIV/0!`. Text divisors
  such as `1/"x"` become `#VALUE!`. Concat `&` uses both operand types so a
  right-hand error (e.g. `"x"&#REF!`) is preserved.
- Formula arithmetic `^` propagates error operands (`2^#REF!` / `#REF!^2` stay
  `#REF!`) instead of Math.pow on coerced zeros.
- `TokenPrecedence` includes `=` so `1=2+3` is RPN `1 2 3 + =` (false), not a
  broken compare-before-add stack that returned the sum.
- `TestCriteria` peels two-char `>=` / `<=` / `<>` before single-char `=`/`</>`;
  COUNTIF/SUMIF/D* multi-char criteria now match numerically (was legacy
  text-branch quirk).
- `IF` preserves condition error types (`IF(1/0,…)` → `#DIV/0!`, `IF(NA(),…)` →
  `#N/A`) instead of collapsing every non-n/b condition to `#VALUE!`.
- Formula-ref Offset/Adjust/Replace re-emit string/sheet tokens via
  `quoteFormulaString`: apostrophe payloads use `'…''…'`, double both quote
  characters so reparse survives the shared quote lexer class.
- VLOOKUP/MATCH/HLOOKUP propagate error lookup keys (`VLOOKUP(1/0,…)` →
  `#DIV/0!`) instead of scanning and returning `#N/A`.
- Named values that collide with FunctionList (`name define N/SUM/PI =…`) win
  on zero-arg RPN (bare name and empty `()` share the same stack shape); range
  calls like `SUM(A1:A1)` still use the builtin. Without a sheet name, arity
  rules stay: `PI` is builtin, bare `N`/`SUM`/`AND()` still FunctionArgsError.
- Formula literals `#NAME?` and `#N/A` lex as specials (SpecialConstants +
  specialvalue completion); `#N/A` added to SpecialConstants.
- `OperandAsNumber` maps non-numeric text/`""` to `e#VALUE!` (not empty type);
  `OperandAsCoord`/`OperandAsType` empty stack returns soft `e#REF!` instead of
  throwing.
- `FormatNumber_defaultCurrency` key used for `currency_char` default (was dead
  `FormatNumber_DefaultCurrency`); bare `$` / empty `[$]` honor `currency_char`
  at format time without poisoning the format cache.
- `ConstantsSetImagePrefix` rewrites both `images/sc_` concat prefixes and
  `images/sc-…` style URLs; `s_fdef_DMIN` says minimum; `s_calcerrnumericnan`
  grammar fixed.
- NPV: blank/text cashflows occupy a discount period as 0 (no silent period
  shift); errors still short-circuit.
- Sort: `OffsetFormulaCoords` optional band freezes external relative refs
  outside the sorted rectangle (`Z1` stays `Z1`); in-band relatives still move.
- IRR blank/text cashflows occupy discount periods as 0 (mirror NPV); DPRODUCT
  zero-match returns 0 not product identity 1.
- StoreIoEventFormula Input: null formDataViewer no-ops so headless TEXTBOX does
  not abort recalc.
- Percent formats that scale into scientific notation keep scaled sci + `%`
  (no longer early-return raw unscaled value).
